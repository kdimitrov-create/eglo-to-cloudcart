# EGLO -> CloudCart XML feed

Scrapes product data from [eglo.com/bg](https://www.eglo.com/bg/) for a given list
of product codes and generates a CloudCart-compatible `<products>` XML feed.

## Output

[`output/eglo_cloudcart_feed.xml`](output/eglo_cloudcart_feed.xml) — the
generated feed, ready for CloudCart import.

For each product the feed contains:

- title, SKU, EAN, price, manufacturer
- description (from the "Описание на продукта" tab)
- gallery images (full size, ordered by `isMain` then `position`)
- characteristics (`<category_properties>`) merged from "Продуктови детайли"
  and the Размери / Техническа информация / Допълнителна информация tabs
- download files (instruction PDF, technical data sheet, safety notes)
- raw tab HTML inside `<tabs>` for client-side rendering

## Usage

```bash
pip install -r requirements.txt

# 1. Edit src/eglo_to_cloudcart.py and set PRODUCT_CODES to the codes you want
# 2. Download product HTML into ./cache/
python src/fetch_products.py

# 3. Parse cache and regenerate output/eglo_cloudcart_feed.xml
python src/eglo_to_cloudcart.py
```

## Layout

```
src/
  eglo_to_cloudcart.py  - HTML parser + XML builder (entry point)
  fetch_products.py     - downloads product pages into ./cache/
cache/                  - scraped HTML (git-ignored, regenerated by fetch)
output/
  eglo_cloudcart_feed.xml - generated CloudCart feed (committed)
```

## Notes

- Codes that don't exist on eglo.com/bg are skipped and listed at the end of
  the run (the search page returns "Не бяха намерени точни резултати").
- Prices are taken from the JSON-LD block (EUR, **VAT included**). Divide by
  1.20 if the target CloudCart store stores net prices.
- Categories are not present in the raw HTML (Magento renders breadcrumbs via
  JS). The scraper falls back to a slug-derived guess; a proper mapping should
  be added before scaling to the full 8000-product catalog.
